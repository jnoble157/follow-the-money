# Profiles build-time manifest

The `officials_manifest.json` in this directory is generated by
`scripts/ingest/build_officials_manifest.py`.

Run it from the repo root after any of these events:
- `scripts/ingest/build.py` has been re-run (new Parquet data)
- A new official is added to `officials_map.json`
- Recipient or filer name mappings need correction

```bash
python scripts/ingest/build_officials_manifest.py
```

The script requires `duckdb` (available in the repo venv) and the Parquet
files under `data/parquet/`. It queries Austin + TEC contributions, merges
per-slug aggregates, and writes the JSON that `registry.ts` imports at build
time.

Commit the regenerated `officials_manifest.json` so the web build stays
deterministic and does not depend on Python or Parquet at Next.js build time.

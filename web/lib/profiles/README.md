# Profiles build-time manifests

The `officials_manifest.json`, `official_details_manifest.json`,
`donors_manifest.json`, and `donor_details_manifest.json` files are generated by
`scripts/ingest/build_manifests.py`. The `official_bios_manifest.json` file is
a tracked OpenAI-generated cache built by `scripts/build_official_bios.ts`.

Run it from the repo root after any of these events:
- `scripts/ingest/build.py` has been re-run (new Parquet data)
- A new official is added to `officials_map.json`
- Recipient or filer name mappings need correction

```bash
python scripts/ingest/build_manifests.py
```

The script requires `duckdb` (available in the repo venv) and the Parquet
files under `data/parquet/`. It queries Austin + TEC contributions, writes the
official roster and official backlink rows, writes a donor summary for list
pages, and writes organization donor detail rows with source-row citations for
each rendered amount.
By default it writes the top 2,200 donor summaries by total; use `--limit` to
build a smaller or larger donor roster.

To backfill missing profile bios for the top officials by reported total, run:

```bash
npm run build:official-bios -- --limit 100
```

The script reads `.env` directly:

```bash
OPENAI_API_KEY=...
OPENAI_PROFILE_BIO_MODEL=gpt-5-mini
OPENAI_PROFILE_BIO_WEB=auto
OPENAI_PROFILE_BIO_REASONING=low
OPENAI_PROFILE_BIO_LIMIT=100
```

It asks the model for a two- or three-sentence bio, accepts only high-confidence
identity matches, rejects numeric claims, and retries with OpenAI `web_search`
only when the no-search pass cannot produce a high-confidence result. Use
`--refresh` to rebuild the cache from scratch and `--web off` to disable the
search retry.

Commit `official_bios_manifest.json` when the generated bios have been reviewed.
Keep `officials_map.json` tracked; it is the small curated input that controls
manual official mappings.
